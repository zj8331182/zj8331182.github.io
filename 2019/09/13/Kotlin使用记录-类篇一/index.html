次构造函数

类也可以声明前缀有 constructor的次构造函数

需要注意的有：

  1. 如果类有一个主构造函数，每个次构造函数需要委托给主构造函数， 可以直接委托或者通过别的次构造函数间接委托。委托到同一个类的另一个构造函数用 this 关键字即可。

  2. 初始化块中的代码实际上会成为主构造函数的一部分。委托给主构造函数会作为次构造函数的第一条语句，因此所有初始化块中的代码都会在次构造函数体之前执行。也就是说上面的代码会先打印“Init block”之后打印“Constructor”

  3. 如果一个非抽象类没有声明任何（主或次）构造函数，它会有一个生成的不带参数的主构造函数。构造函数的可见性是 public。

继承

基础

Kotlin中的class默认是final的，也就是不可被继承的，如果想要声明一个可被继承的超类，需要在class前添加关键字open，同样的类中的属性（包括成员变量和方法）同样是默认final的，如果可被继承需要添加open

构造函数覆盖

如果派生类有一个主构造函数，其基类型可以（并且必须） 用基类的主构造函数参数就地初始化。

如果派生类没有主构造函数，那么每个次构造函数必须使用 super 关键字初始化其基类型，或委托给另一个构造函数做到这一点。 注意，在这种情况下，不同的次构造函数可以调用基类型的不同的构造函数，让我们一起看一下下面的例子

  1. 子类必须覆盖父类的主构造函数，次构造函数必须覆盖主构造函数。
  2. 子类如果有主构造函数，必须在主构造函数中包含父类的主构造函数。
  3. 子类如果没有主构造函数必须在次构造函数中覆盖父类的主构造函数，使用super来调用父类的构造函数。
  4. 子类中重载实现多个父类的构造函数。

  6. 次构造函数和主构造函数的参数表不能相同（构造函数其实就是和类名一个名字的方法，参数表不能相同是基本概念）

属性覆盖

我们这里说的属性包括成员变量和方法，由于这部分二者基本没有区别，就放到一起讲了。

  • 之前我们讲到kotlin中属性默认是final的，如果希望属性可以被重写那么需要在声明时前面加open。
  • 那子类重写属性时需要在声明前加override关键字，表示要重写同名的属性。
  • 如果使用了同名的属性，但是没有在父类中加open或者没有在子类中加override的话都是会报错的欧。
  • 如果子类中重写了某个属性后不希望子类的子类再去重写的话可以再使用final修饰该属性。
  • 如果父类的属性是val子类可以使用var覆盖，但是如果父类是var子类不允许使用val覆盖

类型系统

既然聊到了继承那么我们就说一下kotlin中的类型系统

  • 首先kotlin中也存在java中的Object类型，即是所有类型的父类，这个类型在Kotlin中是Any，不论Int，Long还是我们自己定义的类都会默认继承自Any。
  • Kotlin中不光有所有类的父类，同样有所有类型的子类，Nothing类型，这个类型是在一个方法中发生异常时系统为方法返回的对象的类型
  • Kotlin中可空类型为非可控类型的父类，也就说Any是Any?的子类
  • null为所有可空类型的子类，也就是说可空类型至少有两个子类，一个是null，一个是对应的非可空类型。

抽象类

kotlin中同样有abstract关键字，用法基本与java相同，唯一需要说明的是abstract修饰的类默认就是open的不需要额外再添加open
